# ch03-2 실습과제1

## 문제1
C++에서 생성자를 통하여 멤버변수의 초기화를 하는 이유를 설명하라.

## 답
프로그래머는 사람이기 때문에 멤버변수에 직접 대입을 할 때,<br> 
초기화를 하지 않아 쓰레기값(메모리에 남아있던 아무런 값)이 사용되거나 의도와는 다른 값을 넣게 되는 등의 실수를 할 수 있다.<br>
하지만 생성자를 통한 초기화는 객체가 생성되는 시점에 함수가 자동으로 호출되는 방식이기 때문에 사람이 하는 실수나 오류를 방지할 수 있다.<br> 

## 문제2
생성자 작성시 생성자 몸체에서 멤버를 초기화하는 것보다 멤버 초기화 리스트에서 초기화하는 이유를 설명하라.

## 답
생성자가 호출되면 멤버 초기화 리스트에서 함수 몸체 순서로 실행이 된다. <br>
생성자 몸체에서 멤버를 초기화하는 것보다 멤버 초기화 리스트에서 초기화하는 것이 실행 속도가 빨라 성능을 향상시키기 때문이다. <br>

## 문제3
위임, 타겟 생성자로 나누어 작성하는 이유를 설명하라.

## 답
위임 생성자는 여러 생성자에 중복된 초기화 코드를 하나의 타겟 생성자로 통합한다.<br> 
따라서 위임, 타겟 생성자로 나누어 작성하는 이유는 코드의 중복을 줄이기 위해서이다. <br>

## 문제4
함수 몸체(body)에서 생성자를 직접 호출하면 어떻게 되는지 조사해보라.

## 답
생성자가 호출되어 새로운 객체(임시 객체)가 메모리에 잠시 만들어졌다가, 생성자 호출 문장이 끝나는 즉시 소멸자가 호출되며 소멸한다.<br>
즉, 임시 객체를 만들고 바로 버리는 의미 없는 동작이다.
