# ch06-3 실습과제1

## 문제1
static 멤버변수는 클래스 선언부에 선언하고 외부에서 다시 선언하는 이유를 설명하라.

## 답
static 멤버변수는 객체마다 따로 생성되는 것이 아니라 클래스 전체에서 하나로 공유되는 변수이다.<br>
일반 멤버변수는 객체가 생성될 때마다 객체 안에 메모리가 할당되지만,<br>
static 멤버변수는 객체와 별개로 존재하므로 컴파일러가 실제 저장 공간을 만들 수 있도록 클래스 외부에서 다시 정의해주는 것이다.<br>

## 문제2
static 멤버함수의 클래스 구현부에 static 키워드를 붙이면 안되는 이유를 설명하라.

## 답
static 멤버함수는 클래스 선언부에서 이미 static으로 선언되어 정적 멤버함수임이 결정된다.<br>
따라서 클래스 외부의 구현부에서는 static 키워드를 다시 붙일 필요가 없다. <br>
만약 구현부에서 static을 다시 붙이면, 컴파일러는 이를 클래스의 정적 멤버함수로 해석해야 하는지, <br>
아니면 파일 내부에서만 사용 가능한 일반 정적 함수로 해석해야 하는지 의미를 혼동하게 된다. <br>
