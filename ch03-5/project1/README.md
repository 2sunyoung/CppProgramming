# ch03-5 실습과제1

## 문제1
함수를 사용하는 이유를 설명하라.

## 답 
자주 쓰는 기능을 함수로 정의함으로써 코드의 중복을 방지하고 재활용성을 높일 수 있다. <br>
즉, 코드의 사이즈가 줄고 개발 기간이 단축된다는 장점이 있기 때문에 함수를 사용한다. <br>

## 문제2
짧은 코드를 갖는 함수를 자주 호출하면 발생하는 함수의 오버헤드를 설명하라.

## 답
함수의 오버헤드란 함수를 사용함으로써 발생하는 사용 메모리의 증가 또는 처리시간의 증가를 의미한다.<br>
함수를 실행하는데 걸리는 전체 시간을 T1 + T2 + T3라 한다면, <br>
T2는 함수 몸체에 작성된 코드를 실행하는 데 걸리는 시간이고,<br>
T1과 T3는 함수의 실제 코드가 실행되기 전후에 자동으로 추가된 코드들을 실행하는데 걸리는 시간이라 할 수 있다. <br>
이때 T1 + T3에 해당하는 시간이 바로 함수 호출로 인해 추가적으로 발생하는 오버헤드이다.<br>

## 문제3
12페이지 예제에서 인라인함수가 치환되면 private 멤버변수를 직접 접근하게 된다. 문법오류가 아닌가?

## 답
함수 호출이 올바른 접근인지는 먼저 코드 구조를 기준으로 판단된다.<br>
즉, 함수 안에서 사용된 변수나 기능이 원래 허용된 위치에서 쓰였다면 문제가 없는 것이다.<br>
이때 인라인 함수는 위 설명처럼 이미 허용된 정상 코드를 컴파일 단계에서 단순 치환하는 것이기 때문에<br>
치환된 결과만 보면 private 변수에 직접 접근하는 것 같아 보여도, 문법 오류가 아니라고 할 수 있다.<br>

## 문제4
인라인함수는 소스코드에서는 함수이지만 기계어로 번역되면 함수가 아니다. 이렇게 복잡하게 만든 이유를 설명하라. 다시 말해, 처음부터 함수를 안 쓰면 안되나요?

## 답
함수를 사용해야 캡슐화, 정보 은닉과 같은 객체지향의 장점을 유지할 수 있다.<br>
그러나 아주 간단한 함수를 계속 호출을 하게 되면 오버헤드 문제로 성능 저하가 발생할 수 있다.<br>
이때 인라인 함수를 사용하면 객체지향형 언어의 특징과 성능을 동시에 확보할 수 있다.<br>
